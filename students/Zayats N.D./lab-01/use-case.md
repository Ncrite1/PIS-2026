# Use-case: Поиск фильмов «Что посмотреть?» 🎬

**Первичный актор**: Пользователь 

**Цель**: Найти интересующий фильм для просмотра

**Предусловия**:
- Пользователь имеет доступ к приложению.
- Приложение содержит базу фильмов или подключено к внешнему API.
- У пользователя есть доступ к интернету.

**Основной поток (Happy Path):**
1. Пользователь открывает приложение «Что посмотреть?».
2. Выбирает жанры и категории фильмов.
3. Просматривает список рекомендованных фильмов.
4. Применяет дополнительные фильтры (например: рейтинг, год выпуска, страна).
5. Выбирает понравившийся фильм.
6. Система отображает подробную информацию о фильме (описание, актёры, рейтинг, трейлер).
7. Пользователь нажимает кнопку «Смотреть» или «Добавить в список».
8. Система сохраняет выбор пользователя или перенаправляет на платформу для просмотра.
9. Пользователь получает рекомендации похожих фильмов (опционально).

**Постусловия:**
- Пользователь нашёл фильм для просмотра.
- Выбранный фильм добавлен в «Избранное» или список просмотра (если выбрано).

**Альтернативные потоки:**
- 2a. Если пользователь хочет найти фильм по названию:
    - 2a1. Пользователь вводит название фильма в строку поиска.
    - 2a2. Система отображает результаты поиска.
    - 2a3. Пользователь выбирает фильм и переходит к шагу 6.
- 3a. Если пользователь хочет отсортировать фильмы:
    - 3a1. Пользователь выбирает сортировку (по рейтингу, популярности, дате выхода).
    - 3a2. Система обновляет список фильмов.
    - 3a3. Возврат к шагу 3.
- 7a. Если пользователь хочет добавить фильм в «Избранное»:
    - 7a1. Пользователь нажимает «Добавить в избранное».
    - 7a2. Система сохраняет фильм в списке пользователя.
    - 7a3. Use-case продолжается с шага 9.

**Исключительные ситуации:**
- 3b. Если по выбранным фильтрам не найдено фильмов:
    - 3b1. Система уведомляет пользователя об отсутствии результатов.
    - 3b2. Система предлагает изменить фильтры.
    - 3b3. Use-case продолжается с шага 2.
- 6a. Если не удалось загрузить информацию о фильме:
    - 6a1. Система отображает сообщение об ошибке.
    - 6a2. Система предлагает повторить попытку.
    - 6a3. При неудаче use-case завершается.
- 8a. Если внешний сервис просмотра недоступен:
    - 8a1. Система уведомляет пользователя о проблеме.
    - 8a2. Система предлагает альтернативные платформы (если доступны).
    - 8a3. Use-case завершается неудачей или продолжается при выборе альтернативы.
