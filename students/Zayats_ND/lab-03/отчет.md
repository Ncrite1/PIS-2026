## Ход выполнения работы

### 1. Value Objects (Ценностные Объекты)

**Созданные Value Objects:**

1. **MovieTitle** – *название фильма*

   * **Назначение:** хранит название фильма, используется в сущности Movie
   * **Валидация:**

     * не пустая строка
     * длина < 200 символов
   * **Иммутабельность:** ✅
   * **Файл:** `domain/value_objects/MovieTitle.java`

2. **Genre** – *жанр фильма*

   * **Назначение:** определяет жанр (боевик, драма, комедия и т.д.)
   * **Валидация:**

     * не пустая строка
     * только из списка допустимых жанров
   * **Иммутабельность:** ✅
   * **Файл:** `domain/value_objects/Genre.java`

3. **Rating** – *рейтинг фильма*

   * **Назначение:** хранит числовую оценку фильма (например, IMDb)
   * **Валидация:**

     * > = 0 и <= 10
     * не `null`
   * **Иммутабельность:** ✅
   * **Файл:** `domain/value_objects/Rating.java`

4. **Description** – *краткое описание фильма*

   * **Назначение:** текстовое описание сюжета или аннотация
   * **Валидация:**

     * длина < 1000 символов
     * не `null`
   * **Иммутабельность:** ✅
   * **Файл:** `domain/value_objects/Description.java`

5. **Email** – *email пользователя*

   * **Назначение:** хранит электронную почту для идентификации пользователя
   * **Валидация:**

     * соответствует regex для email
     * не `null`
   * **Иммутабельность:** ✅
   * **Файл:** `domain/value_objects/Email.java`

6. **UserName** – *имя пользователя*

   * **Назначение:** хранит имя или никнейм пользователя
   * **Валидация:**

     * длина > 0 и < 50 символов
     * не `null`
   * **Иммутабельность:** ✅
   * **Файл:** `domain/value_objects/UserName.java`

7. **WatchlistEntryDate** – *дата добавления фильма в избранное*

   * **Назначение:** фиксирует момент, когда пользователь добавил фильм в Watchlist
   * **Валидация:**

     * не `null`
     * дата <= текущая дата
   * **Иммутабельность:** ✅
   * **Файл:** `domain/value_objects/WatchlistEntryDate.java`

### 2. Entities (Сущности)

**Созданные Entity:**

### 1. **Movie** – *фильм*

* ID поле: `id`
* Бизнес-правила:

  * название не может быть пустым
  * рейтинг должен быть от 0 до 10
  * жанр обязателен
* Файл: `domain/model/Movie.java`

### 2. **User** – *пользователь*

* ID поле: `id`
* Бизнес-правила:

  * email должен быть валидным
  * имя пользователя не может быть пустым
* Файл: `domain/model/User.java`

### 3. **Watchlist** – *список наблюдения*

* ID поле: `id`
* Содержит список фильмов пользователя
* Бизнес-правила:

  * один фильм не может добавляться дважды
  * дата добавления фиксируется при создании записи
* Файл: `domain/model/Watchlist.java`

### 4. **Recommendation** – *рекомендация*

* ID поле: `id`
* Связь с пользователем и фильмом
* Бизнес-правила:

  * рекомендация генерируется только для активного пользователя
  * текст рекомендации не может быть пустым
* Файл: `domain/model/Recommendation.java`

### 5. **MovieAggregate** – *агрегат фильма*

* Объединяет Movie + отзывы + рейтинг
* Файл: `domain/model/aggregates/MovieAggregate.java`

### 6. **UserAggregate** – *агрегат пользователя*

* Объединяет User + watchlist + рекомендации
* Файл: `domain/model/aggregates/UserAggregate.java`

### 7. **WatchlistAggregate** – *агрегат списка наблюдения*

* Объединяет Watchlist + добавленные фильмы
* Файл: `domain/model/aggregates/WatchlistAggregate.java`

**Пример кода** (одна Entity):
```java
package com.model;

import com.events.MovieCreatedEvent;
import com.value_objects.*;

import java.util.ArrayList;
import java.util.List;

public class Movie {

    private final Long id;
    private final MovieTitle title;
    private final Genre genre;
    private final Rating rating;
    private final Description description;

    private final List<Object> events = new ArrayList<>();

    public Movie(Long id, MovieTitle title, Genre genre, Rating rating, Description description) {
        if (id == null) throw new IllegalArgumentException("ID required");

        this.id = id;
        this.title = title;
        this.genre = genre;
        this.rating = rating;
        this.description = description;

        events.add(new MovieCreatedEvent(id));
    }

    public MovieTitle getTitle() {
        return title;
    }

    public List<Object> getEvents() {
        return events;
    }

    public Genre getGenre() {
        return genre;
    }

    public Rating getRating() {
        return rating;
    }

    public Long getId() {
        return id;
    }

    public Description getDescription() {
        return description;
    }
}
```

### 3. Aggregate Root (Корневой агрегат)

**Границы агрегата:**
- Корень: MovieAggregate
- Внутренние сущности: Movie
- Value Objects: MovieTitle, Genre, Rating, Description

**Инварианты агрегата:**

| № | Инвариант                                                | Как проверяется                  |
| - | -------------------------------------------------------- | -------------------------------- |
| 1 | **ID фильма не может быть null**                         | В конструкторе `Movie`           |
| 2 | **Название фильма не может быть пустым**                 | В `MovieTitle`                   |
| 3 | **Рейтинг должен быть от 0 до 10**                       | В `Rating`                       |
| 4 | **Жанр обязателен**                                      | В `Genre`                        |
| 5 | **При создании создаётся MovieCreatedEvent**             | В конструкторе `Movie`           |
| 6 | **Нельзя обновить удалённый фильм**                      | В `MovieAggregate.update()`      |
| 7 | **Удаление фильма переводит его в неактивное состояние** | В `delete()`                     |
| 8 | **Любое обновление генерирует событие**                  | `MovieUpdatedEvent` в `update()` |


**Пример кода Aggregate Root:**
```java
package com.model.aggregates;

import com.events.DomainEvent;
import com.events.UserRegisteredEvent;
import com.events.UserUpdatedEvent;
import com.exception.DomainException;
import com.value_objects.Email;
import com.value_objects.UserName;

import java.util.ArrayList;
import java.util.List;

public class UserAggregate {

    private final Long id;
    private Email email;
    private UserName name;

    private final List<DomainEvent> events = new ArrayList<>();

    public UserAggregate(Long id, Email email, UserName name) {

        if (id == null) throw new DomainException("User id required");

        this.id = id;
        this.email = email;
        this.name = name;

        events.add(new UserRegisteredEvent(id));
    }

    public void update(Email email, UserName name) {
        this.email = email;
        this.name = name;

        events.add(new UserUpdatedEvent(id));
    }

    public Long getId() {
        return id;
    }

    public Email getEmail() {
        return email;
    }

    public UserName getname() {
        return name;
    }

    public List<DomainEvent> getEvents() {
        return events;
    }
}
```

## Контрольные вопросы

1. **В чём отличие Value Object от Entity?**
    - **Value Object** определяется *значением* и не имеет собственного идентификатора.  
    - **Entity** определяется *личностью* (ID) и сохраняет свою идентичность даже при изменении состояния.

2. **Почему Aggregate Root должен инкапсулировать доступ к внутренним сущностям?**
    - Чтобы гарантировать соблюдение **инвариантов агрегата** и не позволить внешнему коду изменять состояние напрямую.Все изменения проходят через методы Aggregate Root, что обеспечивает целостность данных и корректные переходы состояний.

3. **Какая роль Domain Events? Приведите пример из вашей системы.**
   - Domain Events фиксируют **значимые изменения в доменной модели**, чтобы другие части системы могли реагировать на них, не нарушая изоляцию домена.

4. **Как вы проверяете инварианты в вашем агрегате? Приведите пример.**
   - Инварианты проверяются в **конструкторе** и **методах изменения состояния**.
    Пример:  
    В методе `complete()` проверяется, что текущее время позже `slot.end()`, иначе бросается исключение: "Cannot complete booking before it ends"

5. **Почему Value Objects делаются иммутабельными?**
   - Потому что они представляют **значения**, а не сущности. Иммутабельность гарантирует, что значение не изменится неожиданно, что делает модель предсказуемой, безопасной и удобной для использования в инвариантах и сравнениях.
